# InkSight 开发背景与展望 (Roadmap)

## 开发背景 (Background)

在信息爆炸的时代，我们每天接触大量的 PDF 报告、电子书和技术文档。然而，传统的阅读工具往往止步于“阅”，缺乏“读”与“思”的深度结合。
- **痛点 1**: 阅读与笔记分离。在 PDF 阅读器中高亮，在 Notion/Obsidian 中做笔记，两者割裂，难以回溯。
- **痛点 2**: 知识碎片化。高亮的内容散落在不同的文档中，难以建立联系。
- **痛点 3**: 缺乏可视化。线性的文字笔记难以表达复杂的逻辑结构。

**InkSight** 应运而生。它的核心理念是 **"Read, Connect, Visualize"**。它不仅仅是一个阅读器，更是一个辅助思考的第二大脑工作台。

## 愿景 (Vision)

打造最流畅的 **"阅读 -> 思考 -> 输出"** 一体化工作流。让知识不再是孤岛，而是连接成网。

## 发展路线图 (Roadmap)

### Phase 1: 基础建设 (Current)
- [x] 支持 PDF, EPUB, TXT, MD 格式。
- [x] 实现基础的高亮与矩形截图。
- [x] 实现阅读器与思维导图的拖拽交互。
- [x] 实现双向链接跳转。

### Phase 2: 体验优化 (Next Steps)
- [x] **全格式标注支持**: 已支持 PDF/EPUB/TXT/MD 的基础高亮与引用。
- [x] **性能优化**: 引入 ELK 算法优化导图布局性能。
- [ ] **移动端适配**: 优化触摸操作，支持 iPad/Android 平板。

### Phase 3: 智能增强 (Future)
- [ ] **AI 辅助阅读**: 集成 LLM，支持文档摘要、划词解释、自动生成思维导图。
- [ ] **云端同步**: 支持 WebDAV 或 S3，实现多设备进度同步。
- [ ] **插件系统**: 允许开发者扩展新的阅读器格式或导图节点类型。
